 Разработать прилагу-каталог
 На твое усмотрение - либо каталог фильмов, либо каталог рецептов
 Все должно быть на одной динамической странице

 Функциональные требования:
 1. Динамический поиск по каталогу (т.е. отображение новых результатов
 поиска при вводе каждого нового символа в строку поиска)
 2. Добавление новых позиций в каталог 
 3. Удаление позиций из каталога
 4. Редактирование существующих позиций

 Каждая позиция в каталоге состоит заголовка и описания (это минимум, можешь добавить поля по своему усмотрению)
 Надо проектировать так, чтобы можно было легко добавить новые поля 
 Для начала хватит двух полей, потом еще к каждой позиции будут картинки, 
 это тоже надо учитывать при проектировании

Технические требования:
1. Использовать js модули (тебе надо будет для этого поставить расширение для vs кода - лайв сервер или что-то такое, не помню точное название)
2. Данные должны хранить в отдельном модуле, у которого должны быть методы для работы с этими данными - получение
всего списка, получение позиции по поиску, редактирование, удаление и тд
3. Объекты в программе (рецепты или фильмы) должны быть оформлены в виде классов

Используем рецепты или фильмы для того, чтобы по окончании этой реализации
можно было перейти с хранения данных в джс модуле на использование АПИ реального сервиса, но это уже отдельно потом
